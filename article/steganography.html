仕組みMechanism

さて今回使用したメッセージを忍び込ませる方法について話していこうと思います。仕組みは意外と単純です。流れは次のようです。まずメッセージを用意したら、それを0と1で構成されたビット列に直します。そうして得られたビット列を画像の(R,G,B)配列の末端に対応させるだけです。各色素は0から255の256段階ですから、変わらないか、変わったとしても1だけです。もちろん画像の配色にもよりますが大抵の画像では大した変化ではありません。

Now, let me talk about the method I used to embed the message. The mechanism is surprisingly simple. The flow is as follows. First, prepare the message, then convert it to a bit string consisting of 0s and 1s. Then, just map that bit string to the least significant bits of the image's (R, G, B) array. Each color component ranges from 0 to 255 in 256 steps, so it either doesn't change or changes by only 1. Of course, it depends on the image's color scheme, but in most images, the change is not significant.

少しわかりにくかったかもしれませんので、具体例を用いて説明します。例えばメッセージが A という文字だった場合、Unicodeコードポイントは65です。これを2進数に直すと、01000001なので、これを画像に忍び込ませることになります。そして、画像の配列が(98, 100, 34), (99, 101, 30), (60, 100, 40)・・・と続いている場合、これらを二進数に直した後、1の位を先ほどの01000001に対応させていきます。8ビット分の情報なので、変わる可能性があるのは3ピクセル目の真ん中の値(100)までです。そうすると(98, 101, 34), (98, 100, 30), (60, 101, 40)・・・と変更されるという感じです。なお、デコード(読み取り)を考えると、一文字分の大きさは統一した方が良さそうです。そのため、今回は8ビットの0詰めで表現しています。読み取りの際はここに注意してください。このような事情で今回は英語のメッセージですが、ここを変更してやると日本語でも送れるはずです(たぶん)。また今回はしていませんが、送るメッセージ自体を暗号化してやれば、一気にそれっぽくなりますね!

The explanation above might have been a little unclear, so I will explain it using a concrete example. Suppose the message is the character "A". Its Unicode code point is 65, which is 01000001 in binary. We embed this bit sequence into the image. Assume that the pixel array of the image looks like (98, 100, 34), (99, 101, 30), (60, 100, 40), ... First, convert these numbers into binary. Then replace the least significant bit of each value so that it corresponds to the message bits (01000001). Since the message consists of 8 bits, the values up to the middle value (100) of the third pixel may change. As a result, the array becomes something like (98, 101, 34), (98, 100, 30), (60, 101, 40), ... When decoding the message, it is better to keep the size of each character consistent. For this reason, I represent each character using 8 bits, padding with zeros on the left. Please keep this in mind when decoding. Because of this limitation, the message used here is written in English. However, by modifying the encoding method, it should also be possible to encode Japanese text (probably). Also, although I did not implement it this time, encrypting the message before embedding it would make the technique much more interesting.

コード例(python)Code Example (Python)

ここまであたかも、私が実際にコードを書いたかのような顔をしてきましたが、実際はほとんどをGeminiに書いてもらいました。やはり私が書くより洗練されていて悔しい限りです。特に、ビット演算の使い方が非常に上手で勉強になりました。

So far, it may have looked like I wrote the code myself, but in fact Gemini wrote most of it. Its code is much more sophisticated than what I could produce, which is a little frustrating. In particular, I learned a lot from the way it uses bit manipulation.


from PIL import Image

def text_to_bin(text):
    """テキストをバイナリ（ビット列）に変換"""
    return ''.join(format(ord(c), '08b') for c in text)

def bin_to_text(binary):
    """バイナリをテキストに変換"""
    text = ""
    for i in range(0, len(binary), 8):
        byte = binary[i:i+8]
        if len(byte) < 8: break
        text += chr(int(byte, 2))
    return text

def encode_image(img_path, message, output_path):
    # 画像を読み込み、RGBA（またはRGB）に変換
    img = Image.open(img_path).convert('RGB')
    pixels = img.load()
    
    # 終了マーカーとしてヌル文字を付加
    message += "\0"
    bin_msg = text_to_bin(message)
    msg_len = len(bin_msg)
    
    width, height = img.size
    idx = 0
    
    for y in range(height):
        for x in range(width):
            r, g, b = pixels[x, y]
            
            # 各チャンネル(R, G, B)の最下位ビットに1ビットずつ隠す
            new_channels = []
            for channel in [r, g, b]:
                if idx < msg_len:
                    # 最下位ビットを書き換え
                    # (channel & ~1) で最後を0にし、| int(bin_msg[idx]) でビットをセット
                    new_val = (channel & ~1) | int(bin_msg[idx])
                    new_channels.append(new_val)
                    idx += 1
                else:
                    new_channels.append(channel)
            
            pixels[x, y] = tuple(new_channels)
            
            if idx >= msg_len:
                img.save(output_path)
                print(f"保存完了: {output_path}")
                return

def decode_image(img_path):
    img = Image.open(img_path).convert('RGB')
    pixels = img.load()
    
    bin_data = ""
    width, height = img.size
    
    for y in range(height):
        for x in range(width):
            r, g, b = pixels[x, y]
            for channel in [r, g, b]:
                # 最下位ビットを取り出す
                bin_data += str(channel & 1)
                
                # 8ビット貯まるごとにヌル文字（終了判定）をチェック
                if len(bin_data) % 8 == 0:
                    current_text = bin_to_text(bin_data[-8:])
                    if current_text == "\0":
                        return bin_to_text(bin_data[:-8])
    return bin_to_text(bin_data)

# --- 実行例 ---
if __name__ == "__main__":
    # 1. メッセージを隠す
    # your/image.png という画像を用意してください

    message = "your message"
    encode_image("your/image.png", message, "encoded_image.png")
    
    # 2. メッセージを取り出す
    decoded_text = decode_image("encoded_image.png")
    print(f"解読されたメッセージ: {decoded_text}")
