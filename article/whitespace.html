はじめにIntoduction

この記事では、whitespaceという難解言語のインタプリタをpythonで作ってみたので、それを紹介しようと思います。whitespaceはそのプログラムがスペースとタブ、そして改行のみで構成されているのが最大の特徴です。そのため、コードを見ただけでは何も書いているようには見えません。このページの最後にHello, World!を出力させるサンプルコードを載せています。見ると色々思う所はあると思うのですが、これでちゃんと動くのってめちゃくちゃかっこよくないですか？？ 苦労して情報に辿り着くというこのサイトの理念にもピッタリです。それに、一見意味のないように思える場所から情報を抜き出すのは、なんだか科学者に必要な態度のような気がして、その意味でも一目惚れした言語です。実はインタプリタはいくつも配布されているのですが、それらがCやpythonで書かれているようでしたので、それは自分で作るしかないだろうと思い至り、頑張って書きました。是非、遊んでいただけると嬉しいです。

In this article, I introduce an interpreter for the esoteric programming language Whitespace that I implemented in Python. The most distinctive feature of Whitespace is that its programs consist only of spaces, tabs, and line breaks. As a result, when you look at the code, it appears as if nothing is written at all.

At the end of this page, I have included a sample program that outputs “Hello, World!”. You may have various reactions when you see it, but isn’t it incredibly cool that something like that actually works? It also fits perfectly with the philosophy of this site—reaching information only after some effort. Extracting information from places that seem meaningless at first glance somehow feels like the kind of attitude scientists should have, and that was another reason why I immediately fell in love with this language.

In fact, several interpreters for Whitespace have already been distributed, but since many of them are written in C or Python, I figured I should try writing one myself. So I went ahead and implemented it. I would be very happy if you try playing with it.

If any readers here belong to the rather extreme camp that believes “high-level languages are for the weak” (of course, I mean that as a compliment), please try writing your own program. And if you do, would you consider sharing it?

It may sound strange coming from me, but honestly, writing programs in this language is not a sane activity. Debugging is extraordinarily difficult and genuinely frustrating. However, it will probably allow you to spend a very luxurious, elegant, and utterly pointless amount of time. To all the extremists out there—I look forward to seeing what you create!

Whitespace 命令体系リファレンス Whitespace Instruction Reference

Whitespaceの命令は、IMP（Instruction Modification Parameter）でカテゴリを選択し、それに続くコマンドで具体的な動作を決定します。凡例：S=Space, T=Tab, L=Linefeed

Whitespace instructions consist of an IMP (Instruction Modification Parameter) followed by a command. Legend: S=Space, T=Tab, L=Linefeed

IMP Command Param 内容Description
S
(Stack)		 S [Num] 数値をスタックに積む Push the number onto the stack
L S-トップを複製Duplicate top item
T S[Num]n番目の値をトップにコピーCopy n-th item to top
L T-トップ2つを入れ替えSwap top two items
L L-トップを捨てるDiscard top item
T L[Num]トップを残し下n個を捨てるSlide n items off
T S
(Math)		 S S-加算 (+)Addition
S T-減算 (-)Subtraction
S L-乗算 (*)Multiplication
T S-整数除算 (/)Integer Division
T T-剰余 (Mod)Modulo
T T
(Heap)		 S-保存 (addr, valをpopして格納)Store in heap
T-取得 (addrをpopして値をpush)Retrieve from heap
L
(Flow)		 S S[Lbl]ラベル定義Mark label
S T[Lbl]サブルーチン呼び出しCall subroutine
S L[Lbl]無条件ジャンプJump unconditionally
T S[Lbl]0ならジャンプJump if zero
T T[Lbl]負ならジャンプJump if negative
T L-サブルーチン終了End subroutine
L L-プログラム終了End program
T L
(I/O)		 S S-文字出力Output character
S T-数値出力Output number
T S-文字入力しヒープへRead character
T T-数値入力しヒープへRead number

注意して欲しいのは、演算はスタックの2個目の値が先で、1個目の値を後に持ってくるということです。つまり引き算を行う命令TSSTがなされた場合、python風に書くと、以下のように処理されます。

One thing to be careful about is that arithmetic operations use the second value on the stack first, followed by the first value. In other words, if the subtraction instruction TSST is executed, it is processed roughly as follows (written in a Python-like style). 


    b = stack.pop()
    a = stack.pop()
    stack.append(a-b)

スタックは、データを"後入れ先出し"と言う順序で管理するデータ構造です。積み上げられた本によく似ています。それに対して、ヒープというのはプログラムの動作中に使われる動的に確保、解放されるメモリ領域です。アドレスと値がセットになって保管され、プログラム中でアドレスを参照することで対応する値を取得できます。フロー制御は、プログラムの命令の順序を変えることができます。プログラムは書かれた命令を順にこなしていきますが、条件分岐を行う際や特定の処理を行う処理群をまとめている場合、命令を飛ばしたり、別の命令群を先に実行してから帰ってくる必要があります。フロー制御を組み合わせることで、こういった処理を可能になります。かなりざっくりした説明になってしまいましたが、ここら辺の内容に興味がある方は各自で調べてみてください。

A stack is a data structure that manages data in a “last in, first out” (LIFO) order. It is very similar to a pile of stacked books. In contrast, a heap is a region of memory that is dynamically allocated and released during the execution of a program. Data are stored as pairs of addresses and values, and the program can retrieve a value by referring to its corresponding address.

Flow control allows a program to change the order in which instructions are executed. Normally, a program executes instructions sequentially as they are written. However, when performing conditional branching or when grouping certain operations together, it becomes necessary to skip some instructions or temporarily jump to another block of instructions and then return later. By combining different flow control operations, such behavior becomes possible.
